# Skills

## Available Skills

### /spec
Draft or refine a specification document. Takes a topic and produces structured Markdown in `spec/`.

### /ux
Create or iterate on a UX artifact — user journey, wireframe description, interaction pattern — in `design/ux/`.

### /arch
Draft or refine an architecture document — system design, data model, component diagram — in `design/architecture/`.

### /adr
Record an Architecture Decision Record in `decisions/`. Follows the format: context, decision, consequences.

### /review
Review an existing spec or design document for completeness, consistency, and gaps. Output a list of questions and suggestions.
